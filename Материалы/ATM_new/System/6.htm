Интерфейсы АТМ

Интерфейсы сети ATM можно классифицировать в зависимости от назначения, определяющегося набором обязательных функций.

  • пользователь-сеть (User Network Interface, UNI) – обеспечивает взаимодействие оборудования пользователя с соответствующим ему сетевым узлом;
  • интерфейс сетевого узла (Network Node Interface, NNI) - обеспечивающим взаимодействие сетевых узлов между собой;
  • интерфейс сеть-сеть (Network-Network Interface) между двумя операторами сетей общего пользования.

    Интерфейсы АТМ

    Интерфейсы UNI определяют все параметры соединения между конечным устройством и коммутатором АТМ, процедуры мультиплексирования и демультиплексирования ячеек, поддерживают сигнализацию между конечной станцией и коммутатором АТМ для установления соединения коммутируемого виртуального соединения, осуществляют управление трафиком между конечной станцией и коммутатором АТМ, определяют адресацию конечных устройств и классифицируются на:

    1. NNI общего пользования (Public NNI)- обеспечивает взаимодействие сетевых узлов внутри сети общего пользования, при этом используется протокол межузловой сигнализации B-ISUP (Broadband Integrated Service User Part) или PNNI (Private Network - Network Interface);
    2. NNI ограниченного пользования (Private NNI) - обеспечивает взаимодействие сетевых узлов внутри сети ATM ограниченного пользования, при этом используется протокол межузловой сигнализации PNNI или IISP (Interim Interswitch Signaling Protocol).

    Интерфейсы NNI ATM отличаются набором обязательных функций классифицируются на:

    1. NNI общего пользования (Public NNI)- обеспечивает взаимодействие сетевых узлов внутри сети общего пользования, при этом используется протокол межузловой сигнализации B-ISUP (Broadband Integrated Service User Part) или PNNI (Private Network - Network Interface);
    2. NNI ограниченного пользования (Private NNI) - обеспечивает взаимодействие сетевых узлов внутри сети ATM ограниченного пользования, при этом используется протокол межузловой сигнализации PNNI или IISP (Interim Interswitch Signaling Protocol).

    Взаимодействие сетей ATM общего пользования, принадлежащих различным операторам, должно осуществляться через интерфейс B-ICI (B-ISDN Inter Carrier Interface, спецификация af-bici-0013.003), созданный Форумом ATM.

    На интерфейсе B-ICI используются:

  • формат ячеек NNI в соответствии с Рекомендацией МСЭ-Т I.361;
  • функции уровня ATM в соответствии со спецификацией Форума АTM UNI 3.0/3.1
  • протокол сигнализации B-ISUP, разработанный МСЭ-Т.

    В качестве физических интерфейсов могут быть использованы ЕЗ STM-1, STM-4.

    Взаимодействие сетей ATM может осуществляться по постоянным или коммутируемым соединениям.

    Интерфейс U1 - взаимодействие узлов магистральной сети в соответствии с требованиями протокола АТМ для интерфейса NNI (интерфейс сеть-сеть), определенными в Рекомендации МСЭ-Т I.361. При этом для организации коммутируемых соединений должна использоваться сигнализация B-ISUP в соответствии с Рекомендациями МСЭ-Т серии Q.27xx.

    При установлении соединений между узлами магистральной сети в соответствии с применяемым протоколом сигнализации B-ISUP выбор маршрута должен осуществляться по постоянными маршрутным таблицам, которые устанавливаются со стороны системы управления сетью (статическая маршрутизация).

    Интерфейс U2 - взаимодействие узлов магистральной сети, принадлежащих различным операторам, в соответствии с требованиями протокола АТМ для интерфейса NNI, определенными в Рекомендации МСЭ-Т I.361. При этом для организации коммутируемых соединений должна использоваться сигнализация B-ISUP в соответствии с Рекомендациями МСЭ-Т серии Q.27xx.

    Интерфейс U3 - взаимодействие узлов магистральной сети с узлами сетей АТМ зарубежных операторов осуществляется в соответствии с требованиями протокола АТМ для интерфейса NNI, определенными в Рекомендации МСЭ-Т I.361. При этом для организации коммутируемых соединений должна использоваться сигнализация B-ISUP в соответствии с Рекомендациями МСЭ-Т серии Q.27xx.

    Интерфейс U4 - взаимодействие узлов магистральной и региональной сети осуществляется в соответствии с требованиями протокола АТМ для интерфейса NNI, определенными в Рекомендации МСЭ-Т I.361. При этом для организации коммутируемых соединений должна использоваться сигнализация B-ISUP в соответствии с Рекомендациями МСЭ-Т серии Q.27xx. Если региональная сеть использует сигнализацию PNNI, то при взаимодействии с магистральной сетью на узле региональной сети необходимо осуществить преобразование сообщений сигнализации B-ISUP в сообщения сигнализации PNNI и обратно.

    На первых этапах создания магистральной сетей АТМ в случае отсутствия региональной сети могут использоваться узлы доступа для подключения сетей ТФОП и сетей ПД (интерфейсы U8 и U9).

    Интерфейс U5 - взаимодействие узлов региональной сети, осуществляется одним из двух способов:

  • в соответствии с требованиями протокола АТМ для интерфейса NNI, определенными в Рекомендации МСЭ-Т I.361. При этом для организации коммутируемых соединений должна использоваться сигнализация B-ISUP в соответствии с Рекомендациями МСЭ-Т серии Q.27xx;
  • в соответствии с требованиями протокола АТМ, определенными в Спецификации Форума АТМ PNNI версии 1.0. При этом для организации коммутируемых соединений должна использоваться сигнализация, определенная в указанной спецификации.

    На региональной сети АТМ могут использоваться постоянные, полупостоянных, комбинированных и коммутируемых соединений АТМ.

    При установлении соединений между узлами региональной сети при использовании протокола сигнализации B-ISUP выбор маршрута должен осуществляться в соответствии с постоянными маршрутными таблицами (статическая маршрутизация). При использовании сигнализации PNNI могут использоваться динамические таблицы маршрутизации, которые устанавливаются в соответствии с текущим состоянием сети (динамическая маршрутизация).

    Интерфейс U6 - взаимодействие узлов региональной сети с узлами служб должно осуществляться одним из следующих способов:

  • в соответствии с требованиями протокола АТМ для интерфейса NNI, определенными в Рекомендации МСЭ-Т I.361. При этом для организации коммутируемых соединений должна использоваться сигнализация B-ISUP в соответствии с Рекомендациями МСЭ-Т серии Q.27xx;
  • в соответствии с требованиями протокола АТМ, определенными в Спецификации Форума АТМ PNNI версии 1.0. При этом для организации коммутируемых соединений должна использоваться сигнализация, определенная в указанной спецификации.

    Интерфейс U7 - взаимодействие узлов егиональных сетей, принадлежащих различным операторам, организуется одним из следующих способов:

  • в соответствии с требованиями протокола АТМ для интерфейса NNI, определенными в Рекомендации МСЭ-Т I.361. При этом для организации коммутируемых соединений должна использоваться сигнализация B-ISUP в соответствии с Рекомендациями МСЭ-Т серии Q.27xx;
  • в соответствии с требованиями протокола АТМ, определенными в Спецификации Форума АТМ B-ICI версии 1.1. При этом допускается использование Спецификации Форума АТМ B-ICI версии 2.0.

    Подключение абонентов сети АТМ должно осуществляться в точках региональной сети, реализующих функции сетевого окончания.

    Функция сетевого окончания может быть либо реализована в оборудовании оконечного или оконечно-транзитного узла сети АТМ, либо вынесена в оборудование, устанавливаемое у абонента.

    Сетевое окончание может включать следующие типы интерфейсов для взаимодействия с оконечным оборудованием пользователя (интерфейс U8):

  • интерфейс UNI (интерфейс пользователь-сеть) в соответствии с Рекомендацией МСЭ-Т I.361. При этом для организации коммутируемых соединений должна использоваться сигнализация DSS2 в соответствии с Рекомендациями МСЭ-Т серии Q.29xx;
  • интерфейс UNI в соответствии со Спецификациями Форума АТМ UNI 3.1. При этом допускается использование интерфейса UNI, требования к которому определены в Спецификации Форума АТМ UNI 4.0;
  • специализированный интерфейс пользователя, соответствующий стандартному интерфейсу для передачи информации различного вида (например локальных сетей, сетей Х.25, Frame Relay, ЦСИС, ТФОП и т.п.), реализующий функцию преобразования формата передаваемой информации в формат протокола АТМ.

    Интерфейс U9 - взаимодействие региональной сети АТМ с другими сетями электросвязи, осуществляется в точках сетевого окончания:

  • по интерфейсу UNI, в соответствии с Рекомендацией МСЭ-Т I.361. При этом для организации коммутируемых соединений должна использоваться сигнализация DSS2 в соответствии с Рекомендациями МСЭ-Т серии Q.29xx;
  • по интерфейсу UNI, в соответствии со Спецификациями Форума АТМ UNI 3.1. При этом допускается использование интерфейса UNI, требования к сигнализации которого определены в Спецификации Форума АТМ UNI 4.0;
  • по специализированному интерфейсу пользователя, соответствующему стандартному интерфейсу для передачи информации различного вида (например локальных сетей, сетей Х.25, Frame Relay, ЦСИС, ТФОП и т.п.), реализующий функцию преобразования формата передаваемой информации в формат протокола АТМ.
