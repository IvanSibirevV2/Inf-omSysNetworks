Адрес DCC AESA

Адреса AESA представляются в шестнадцатеричной форме, и каждый такой адрес имеет размер в 20 байт. Адрес делится на два сегмента: IDP (Initial Domain Part) и DSP (Domain Specific Part), каждый из которых состоит из нескольких полей (рис.1).

Рисунок 1

  • Сегмент IDP определяет тип адреса и тип уполномоченного, который отвечает за управление этим адресом. В этом сегменте есть два поля:
    • однобайтовое поле AFI (Authority and Format Indicator) - для рассматриваемого формата адреса всегда имеет фиксированное значение, равное 39;
    • двухбайтовое поле IDI (Initial Domain Identifier) - содержит код, идентифицирующий страну, в которой зарегистрирован адрес (DCC — Data Country Code).
  • Сегмент DSP также разделен на несколько дополнительных полей:
    • 10-байтовое поле HO-DSP (High Order Domain Specific Part) - идентификация части адресного пространства, которая выделена определенной подсети
    • 6-байтовое поле ESI (End System Identifier — идентификатор конечной системы) - служит для уникальной идентификации конечной системы АТМ в определенной подсети (комбинация полей IDP + HO-DSP + ESI должна быть уникальной);
    • однобайтовое поле SEL (Selector — селектор).

    Поле SEL не влияет на маршрутизацию запросов на установление коммутируемых виртуальных соединений в сети АТМ, поэтому оно может использоваться конечной системой.
